= `acdc-pdf-theme`

Design tokens (colours, typography, spacing) plus embedded free fonts and a syntax-highlight theme, for acdc's PDF output.

This is an internal workspace crate. It is not published to crates.io and has no stability guarantees outside the `acdc-pdf-*` pipeline.

== Where it sits

[source]
....
acdc-pdf-theme  ──▶  acdc-pdf-typst   (preamble reads the tokens)
              └──▶  acdc-pdf-render  (loads the bundled fonts + syntax theme)
....

The crate is *Typst-free*: it carries only data and bundled assets. The tokens drive the preamble that `acdc-pdf-typst` generates; the fonts and syntax theme are loaded by `acdc-pdf-render`.

== What it provides

- **`Theme`** — the design tokens (`palette`, `typography`, `spacing`). Tokens are *data*, loaded from YAML via `Theme::from_yaml_str`, so a caller can supply a brand theme without recompiling.
- **`Theme::default`** — the bundled neutral theme (white page, grayscale, bundled fonts only). Its private embedded YAML remains the single source of truth and is parsed once per process, so the tokens cannot drift from a hand-written struct.
- **`embedded_fonts`** — bytes for every bundled free font face (IBM Plex plus
  Noto Color Emoji).
- **`EMOJI_FONT_FAMILY`** — the family name of the bundled emoji face; `acdc-pdf-typst` appends it to every font stack so emoji render as colour glyphs under any theme.
- **`highlight_theme`** / **`HIGHLIGHT_THEME_PATH`** — the bundled source-code highlighting theme and the virtual path it is served at.

== Customising

Supply a theme YAML file (via the CLI `--theme` flag or `PdfOptions::theme`) that deserialises into `Theme`; omit it to use `Theme::default`. The bundled `assets/theme/default.yaml` documents every available token. The parser does not impose an arbitrary document-size limit. The PDF converter applies a separate 1 MiB file-input limit before parsing. The schema is strict: unknown fields, invalid colours, invalid font stacks, invalid font weights, and non-finite or negative measurements are rejected with field-specific errors. Parsing uses the safe-Rust `serde-saphyr` backend, including its complexity and alias-expansion budgets.

== Bundled asset policy

The bundled fonts retain their upstream license files, and tests parse every
font to confirm its family, style, and weight. The syntax theme is exercised
through its real Typst consumer.

Noto Color Emoji is intentionally retained as an unconditional fallback. Its
25,110,660-byte face accounts for most of the crate's asset size, but guarantees
consistent, offline colour emoji instead of missing-glyph boxes in default PDF
output. The renderer shares its compile-time bytes and parsed face across
renders, so that product choice does not also impose a fresh 25.1 MB heap copy
or parse on every document. Making emoji optional should be reconsidered if
binary download size becomes more important than identical offline output.

== Testing

[source,console]
....
cargo nextest run -p acdc-pdf-theme --all-features
....

Tests parse the bundled default YAML, verify every embedded font, and exercise
the syntax theme through Typst in `acdc-pdf-render`.
