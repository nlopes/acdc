This document is normal HTML content with terminal-style source blocks rendered
as selectable terminal previews when the HTML converter is built with the
`terminal` feature.

The content below is intentionally written like a terminal workflow so the
preview has terminal-shaped material to render.

== Terminal Session

[source,console]
----
$ acdc --version
acdc 0.2.0

$ acdc convert README.adoc --backend html --variant semantic -o /tmp/readme.html
Generated html5s file: /tmp/readme.html

$ acdc convert converters/html/samples/terminal_preview.adoc \
    --backend html \
    -o /tmp/acdc-terminal.html
Generated html file: /tmp/acdc-terminal.html
----

== Output Checklist

The terminal preview should make these details visible as selectable browser
text:

* command prompts beginning with `$`
* wrapped command lines
* generated output paths
* *bold text*, _italic text_, `monospace text`, and [.underline]#underlined text#

Keyboard shortcuts such as kbd:[Ctrl+K] and menu paths such as
menu:File[Export, HTML] should render visibly in both the HTML page and terminal
preview.

== Lists

* Terminal previews should show list bullets.
* Wrapped lines should stay readable inside the configured terminal width.
* Nested content should keep indentation:
** child item one
** child item two

== Admonition

NOTE: The `:acdc-terminal:` attribute is the document-level opt-in. Use
`:acdc-terminal-cols:` / `:acdc-terminal-rows:` to control the terminal grid size, or omit
rows to let acdc size the preview from the rendered text.
Without `:acdc-terminal:`, standard and semantic HTML output stay unchanged.
